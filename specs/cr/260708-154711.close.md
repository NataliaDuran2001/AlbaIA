## Alcance entregado

Endpoint `POST /api/negocio` (acción `generar_documento`) funcional end-to-end, con dos

- **Plantilla hardcodeada** (`solicitud_sociedad_emprendimiento`): sin RAG ni IA.
- **RAG + IA** (contrato de trabajo, escrituras, RTU, etc.): full-text sobre la base legal
  + `deepseek-v4-pro`, con citación de artículos.

## Resumen de implementación

El endpoint y `lib/fases/generarDocumento.ts` ya existían. El trabajo fue **verificación
end-to-end contra el server real**, que destapó **2 bugs reales que se corrigieron**:

1. **Consulta RAG rota** (`generarDocumento.ts:194`): la consulta era
   `"{nombreDoc} para {sociedad|comerciante} en Guatemala"`. Como
   `websearch_to_tsquery` usa semántica AND, exigía que un mismo artículo contuviera
   todas esas palabras → **0 resultados para TODOS los documentos RAG**. Solo la plantilla
   SE (que no usa RAG) funcionaba. Fix: usar solo `nombreDoc` como consulta; el contexto
   del perfil va en el prompt, no en la búsqueda.

2. **Parser JSON frágil** (`generarDocumento.ts`): `deepseek-v4-pro` es un modelo de
   razonamiento que a veces antepone cadena de pensamiento antes del JSON. El parser
   cortaba entre el primer `{` y el último `}`, rompiéndose de forma intermitente
   ("El modelo no devolvió JSON válido"). Fix: helper `extraerPrimerObjeto` que recorta
   el primer objeto JSON balanceado (respetando strings/escapes) + elimina bloques
   `<think>`. Riesgo previsto en `HANDOFF-IA.md`.
